Documentation Search

Keywords: ElasticSearch, Docker, Javascript, markdown, React

While at Capital One, I participated in the Capital One Developer Academy. As our "Dojo" project, we worked with a Capital One team on an application of theirs. The team owned their own component library for Angular and React and used an internally hosted site to enable users to search for documentation. Our job was to take these markdown documentation pages, parse them, and input them into an ElasticSearch engine that enabled users to effectively search for what they needed with a searchbar. On a team of five, we implemented this and enabled autocomplete and fuzzy search. Our next step was to host our search in AWS using Lambda.

This functionality was developed at Capital One with their resources so the source code and demo is private.
